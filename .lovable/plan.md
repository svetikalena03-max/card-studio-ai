## AI Marketplace Card Studio — Frontend

Премиум-минималистичный SaaS-интерфейс в стиле Apple для генерации карточек товаров для маркетплейсов. На этом этапе — только frontend с моковыми данными; интеграция с AI выносится в отдельный сервис-файл, готовый к подключению.

### Дизайн-система
- Светлый фон, много воздуха, мягкие тени, скруглённые карточки
- Акцентный цвет — глубокий индиго/чёрный, нейтральные серые
- Типографика: крупные заголовки, чёткая иерархия (системный sans-serif в духе SF Pro / Inter)
- Компоненты shadcn/ui (Card, Tabs, Button, Input, Select, Textarea, Badge, Toast)

### Структура страниц (TanStack Router)

```
src/routes/
  __root.tsx           — общий layout с шапкой (логотип + навигация)
  index.tsx            — Главная (hero, CTA)
  generate.tsx         — Форма генерации
  result.$id.tsx       — Результат с вкладками
  history.tsx          — История проектов
```

### 1. Главная (`/`)
- Заголовок: **AI Marketplace Card Studio**
- Подзаголовок: «Создавайте карточки товаров, которые отличаются от конкурентов»
- Кнопка **Создать карточку** → `/generate`
- Блок с 3–4 преимуществами (SEO, отстройка, инфографика, Reels)

### 2. Страница генерации (`/generate`)
Форма с полями:
- Название товара (Input)
- Маркетплейс (Select: Wildberries, Ozon, Яндекс Маркет, Lamoda, другое)
- Целевая аудитория (Input)
- Характеристики товара (Textarea)
- Отличия от конкурентов (Textarea)
- Ссылки на конкурентов (Textarea, по строке)
- Желаемый стиль (Select: premium, minimalism, mass market, kids, tech, wellness, home aesthetic)
- Дополнительные пожелания (Textarea)

Кнопка **Сгенерировать карточку** → вызывает сервис, сохраняет результат в localStorage, переходит на `/result/$id`. Состояние loading со скелетоном.

### 3. Результат (`/result/$id`) — карточка с вкладками
Шапка: название товара, маркетплейс, стиль, дата.

Вкладки:
- **Анализ** — Анализ конкурентов, Слабые места, Стратегия отстройки
- **Тексты** — SEO-название, Короткое описание, Полное описание, Преимущества, Для кого товар
- **SEO** — SEO-ключи (chips), Антишаблон
- **Инфографика** — Идеи инфографики (список блоков с текстом)
- **Фото** — Идеи lifestyle-фото, Промты для AI-фото
- **Reels** — Идеи Reels

Действия (кнопки в шапке результата):
- Скопировать результат (всё)
- Скопировать SEO
- Скопировать тексты для инфографики
- Скачать результат (.txt / .md)
- Создать новую карточку → `/generate`

Каждое копирование — toast-уведомление.

### 4. История проектов (`/history`)
- Сетка карточек: название товара, маркетплейс, стиль, дата
- Клик → открывает `/result/$id`
- Пустое состояние с CTA «Создать первую карточку»
- Кнопка удаления карточки

### 5. Сервисный слой AI (заготовка)

```
src/lib/ai/
  openai-service.ts    — функция generateProductCard(input): Promise<CardResult>
                          с заглушкой (мок-ответ через setTimeout) и
                          закомментированным шаблоном fetch к OpenAI API
  types.ts             — типы CardInput, CardResult (все поля результата)
  storage.ts           — сохранение/загрузка истории в localStorage
```

После одобрения плана: при подключении реального OpenAI потребуется добавить API-ключ через секреты Lovable Cloud и перенести вызов в server function (чтобы не светить ключ во фронте). Сейчас — только TODO-комментарий в сервисе.

### Технические детали
- Stack: TanStack Start + React 19 + Tailwind v4 + shadcn/ui (уже в проекте)
- Состояние формы — локальный useState; история — localStorage
- ID карточки — `crypto.randomUUID()`
- Все тексты UI — на русском
